# 2026-04-08 變更紀錄

## 新增
- **classifier/core/field_content_rules.py** — 欄位級內容感知敏感度規則
  - 每個欄位獨立的分析函式，根據**實際值**（非僅欄位名稱）回傳 `HIGH` / `MEDIUM` / `LOW`
  - `ip_address`：內網 → LOW、RFC1918 → MEDIUM、外部 → HIGH
  - `behavioral_score`：≥60 → LOW、40-59 → MEDIUM、<40 → HIGH
  - `failed_attempts`：0-2 → LOW、3-4 → MEDIUM、≥5 → HIGH
  - `login_status`：success → LOW、failed → HIGH
  - `session_duration`：<300 → LOW、300-599 → MEDIUM、≥600 → HIGH
  - `timestamp`：白天 → LOW、夜間 → HIGH
  - `user_id`：一律 HIGH（PII）
  - `device_type` / `anomaly`：一律 LOW
- **tests/test_feature_sensitivity.py** — 20 項測試涵蓋內容感知分析、動態加密決策、MEDIUM 策略、向後相容

## 重構
- **classifier/core/feature_sensitivity.py** — 從靜態欄位名稱分類升級為內容感知分類
  - 新增 `FieldSensitivityLevel` 列舉（HIGH/MEDIUM/LOW）
  - 新增 `FieldAnalysisResult` 資料結構
  - 新增 `analyze_field(name, value)`：單欄位內容分析
  - 新增 `analyze_record(record)`：整筆 record 批次分析
  - 修改 `sensitive_indices()`：依內容動態決定加密欄位，接受 `record_sensitivity` 參數
  - 保留舊 API（`flag_feature`、`classify_features`）完全向後相容
- **adaptive_module/core/mixed_protection.py** — `protect_record()` 新增 `raw_record` 與 `record_sensitivity` 參數，將原始值傳入內容分析器
- **adaptive_routing/adaptive_router.py** — `process()` 傳入 `raw_record` 和 record 敏感等級，啟用動態欄位加密

## 重構
