# Changelogs 撰寫指南

## 檔案命名

每次變更紀錄以獨立檔案存放，檔名格式為：

```
changelogs/YYYY-MM-DD.md
```

若同一天有多次紀錄，加上流水號：

```
changelogs/2026-03-12-2.md
```

## 檔案格式

```markdown
# YYYY-MM-DD 變更紀錄

## 新增
- **模組名稱** — 簡短說明
  - 細項描述

## 重構
- **檔案或模組** — 做了什麼、為什麼

## 修復
- **檔案或模組** — 修復了什麼問題

## 移除
- **檔案或模組** — 移除原因
```

## 分類說明

| 類別 | 使用時機 |
|------|----------|
| **新增** | 新模組、新功能、新測試 |
| **重構** | 重新命名、搬移檔案、抽取邏輯、更新 import 路徑 |
| **修復** | 修正 bug、修正測試失敗 |
| **移除** | 刪除不再使用的程式碼或檔案 |

只列出實際用到的類別，不需要的可以省略。

## 撰寫原則

- 語言使用**繁體中文**
- 每條紀錄以 `**粗體**` 標示影響的模組或檔案
- 先寫「做了什麼」，再寫「為什麼」
- 類別名稱（如 class、method）用反引號 `` ` `` 包起來
- 建立新檔案後，在根目錄的 `CHANGELOG.md` 索引頁加上對應連結

## 更新索引

在 `CHANGELOG.md` 中新增一行連結，最新的放在最上面：

```markdown
- [2026-03-13](changelogs/2026-03-13.md)
- [2026-03-12](changelogs/2026-03-12.md)
```
