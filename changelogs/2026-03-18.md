# 2026-03-18 變更紀錄

## 新增
- **server_module** — 從已訓練的 sklearn LR 模型載入真實 weights 的 Server 端推論模組
  - `TrainedModelServer`：實作 `ModelServerLike` protocol，從 `lr_model.joblib` 提取 `coef_` / `intercept_` 作為權重
  - `from_weights()` classmethod：方便測試時以明確 weights 建立實例
  - `infer(payload)`：支援 plaintext-only 與 mixed plain+Paillier encrypted payload
  - `COLUMN_TO_INTERNAL`：sklearn 欄位名 → 內部欄位名對應表
- **pipeline** — 統一端到端管線模組，串接流量載入到異常判斷的完整流程
  - `PipelineConfig`：整合 routing、threshold、dataset/model 路徑、sample size 等設定
  - `EndToEndPipeline`：組裝 `PaillierEncryptor` + `TrainedModelServer` + `IDSSystem`，逐筆處理 record 並產出 metrics
  - `run_pipeline_demo.py`：CLI demo 腳本，支援 `--sample-size`、`--threshold`、`--seed`、`--output-dir`
  - 輸出 `pipeline_results.json`（metrics + route 分布）與 `pipeline_predictions.csv`（逐筆結果）
- **tests/test_server_module.py** — 7 項測試
  - 從 joblib 載入 weights 正確性
  - `from_weights` classmethod
  - plaintext-only 推論
  - 未知欄位忽略（weight=0）
  - mixed plain+encrypted 推論（真實 Paillier）
  - encrypted 非零權重驗證
  - 真實模型 inference 合理性
- **tests/test_end_to_end.py** — 7 項測試
  - `process_event` 返回預期 keys
  - probability 在 [0, 1] 範圍
  - HIGH 敏感度 → HE 路徑
  - HE 路徑 `decrypted=True`
  - plain 路徑 `decrypted=False`
  - latency 為正值
  - 確認使用 `TrainedModelServer`（非 `ServerLRStub`）
