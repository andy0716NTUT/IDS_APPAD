# 2026-04-16 變更紀錄

## 新增 — Client/Server 分離架構

將原本單一 process 的推論邏輯拆分為獨立的 Client 和 Server 服務，透過 HTTP 通訊。
Server 僅持有模型權重與公鑰，永遠不能解密；私鑰只存在於 Client 端。

### 新增檔案

- **server_module/serialization.py** — CKKS payload 序列化/反序列化工具
  - `serialize_payload()` / `deserialize_payload()`：將 CKKSVector 轉為 base64 JSON wire format
  - `serialize_result()` / `deserialize_result()`：推論結果序列化
- **server_module/server_app.py** — 獨立 Flask 推論 Server（port 5001）
  - `GET /health`：健康檢查
  - `POST /context`：接收 Client 上傳的公開 CKKS context（不含私鑰）
  - `POST /infer`：接收序列化 payload，執行推論，回傳結果
- **server_module/remote_server.py** — Client 端 HTTP Proxy
  - `RemoteLRModelServer`：實作 `ModelServerLike` protocol，透過 HTTP 轉發推論請求
  - 首次遇到加密欄位時自動上傳公開 CKKS context
- **deploy_server/** — 獨立部署目錄，可直接放到 Linux server
  - 包含 `Dockerfile`、`start_server.sh`、`requirements.txt` 與所有必要模組
- **tests/test_serialization.py** — 序列化 round-trip 測試（6 項）
- **tests/test_server_app.py** — Server API 整合測試（7 項）
- **tests/test_remote_server.py** — Remote client mock 測試（5 項）

### 修改檔案

- **ckks_homomorphic_encryption/he_encryptor.py** — 新增 `public_context_bytes()` 方法，匯出不含私鑰的 context
- **adaptive_routing/system.py** — `IDSSystemConfig` 新增 `server_url` 參數，自動建立 `RemoteLRModelServer`
- **pipeline/appad_pipeline.py** — CLI 新增 `--server-url` 參數，主 pipeline 和 privacy ratio sweep 均支援遠端推論
- **requirements.txt** — 新增 `requests>=2.31,<3`
- **logistic_regression_model/inference/inference_tools.py** — `pandas` 改為 lazy import，Server 端不需安裝 pandas

## 修改 — 前端增強

- **frontend/src/app/App.tsx** — 全面改版 IDS 監控儀表板
  - 新增「執行控制面板」：可選推論模式、隨機種子、隱私比例掃描開關
  - 新增「模式比較」分頁：一鍵執行三模式比較，含柱狀圖、雷達圖、流量分布圖、延遲比較圖
  - 新增額外指標列：Precision、Recall、F1 Score、未加密敏感比例、流量結構圓餅圖
  - 執行紀錄新增推論模式彩色標籤
  - Header 顯示遠端/本地推論狀態
  - 系統日誌改為深色終端機風格
- **frontend/backend_api.py** — 新增 `--server-url` CLI 參數與 `GET /api/server-info` 端點
  - `/api/run` 支援 `serverUrl` 參數，自動傳遞給 pipeline

## 修復

- **adaptive_routing/adaptive_router.py** — `time.time()` 改為 `time.perf_counter()`，修復 Windows 上延遲為 0 的問題
- **main.py** — `subprocess.Popen` 加上 `shell=True`，修復 Windows 上 npm 無法啟動的問題
